Firewall Processing Pipeline

Our engine processes thousands of active connections instantly. See how the stateful inspection pipeline secures your boundaries sequentially.

1
Socket Discovery

Identifying active process sockets (TCP/UDP) and mapping raw connections immediately via os-level utilities.

2
Zone Classification

Automatically mapping local and remote addresses against RFC1918 boundaries to determine Internal, Outbound, or Inbound directions.

3
Stateful Analysis

Crucial context building. The engine tracks exact TCP negotiation streams from SYN-SENT to ESTABLISHED, linking the raw socket to its exact process PID and application name.

4
Policy Enforcement

Applying live restriction rules instantly, breaking connections that violate explicit blocklists or default-deny inbound patterns.

5
Continuous Logging

Pushing threat analytics directly to the dashboard canvas, keeping the UI perfectly in sync for real-time monitoring.

System Documentation

CCAF (ConnTrack Context-Aware Firewall) operates completely locally using Python and Flask, deeply integrating with host-level metrics to enforce zero-latency connection rules.

Architecture & Stack

The engine consists of two core components running synchronously to maintain a highly-optimized live state table.

  • Flask API Backend Handles the local dashboard UI and exposes CRUD endpoints for dynamic rule management (`/api/block`, `/api/unblock`).
  • Stateful Logic Engine Located in `core/rule_engine.py`. Monitors connections via OS-level socket APIs (`psutil`) and maintains established TCP states from SYN-SENT through FIN-WAIT.
